Model

Sumar

Modelul este folosit pentru a se customiza editarea unei instanțe. O clasa care nu va fi folosită în interiorul unui proces, trebuie sa fie decorata cu atributul Foundation.Unit.Attributes.ModelBuilder. Modelul este o clasa care mosteneste clasa abstracta Foundation.MVC.Model.

public class Model : Foundation.MVC.Model
{
    protected override void ConfigureViews()
    {
        AddRootView();
        Views.Add(NewsView);
        Views.Add(TagsView);
   }

    public Foundation.MVC.Views.One2NView2 NewsView
    { /* Code */ }
    public Foundation.MVC.Views.N2NView TagsView
    { /* Code */ }
}

View-uri

View-urile intr-un model sunt elemente folosite pentru a se construi layout-ul de editare în pagina de editare. În cele ce urmează va vor fi prezentate cele mai folosite view-uri în Sinapsi.

View-ul NoActionInstanceView

View-ul NoActionInstanceView este folosit pentru a incarca o schema de editare a unei instanțe. Metoda GetRootView() din clasa abstracta Foundation.MVC.Model returnează un view de tipul NoActionInstanceView în care schema de editare este schema de editare predefinita a clasei.

protected Foundation.MVC.Views.NoActionInstanceView GdprView
{
    get
    {
        Foundation.MVC.Views.NoActionInstanceView view = new
                    Foundation.MVC.Views.NoActionInstanceView();
        view.Identification = "GdprAgreement";
        view.VisibleOnNew = false;
        view.EditSchema = "GDPRSchema";
        view.Caption = "Acord GDPR";
        view.Instance = Contact;
        return view;
    }
}

În exemplu precedent putem observa folosirea unor proprietati care au fost setate pentru configurarea view-ului. Valoarea proprietății Identification trebuie sa fie unica în contextul viewer-urilor care fac parte din model.
Proprietatea VisibleOnNew va determina dacă acel view va fi afișat sau nu atunci cand instanta este nouă. Setarea proprietății EditSchema se va face atunci cand pentru NoActionInstanceView nu vrem sa se incarce schema de editare default. Proprietatea Instance este necesara pentru a preciza ce instanța va fi folosită.

View-ul One2N

View-ul One2N (unu-la-mai-mulți) va fi folosit pentru colecțiile unei instanțe.

Exemplu:

O categorie de știri poate fi regăsită în mai multe știri, dar o știre are o singură categorie

public Foundation.MVC.Views.One2NView2 NewsView
{
     get
     {
          Foundation.MVC.Views.One2NView2 view = new 
                       Foundation.MVC.Views.One2NView2();
          view.Identification = "NewsCollection";
          view.Caption = "News";
          view.AllowNew = true;
          view.AllowDelete = false;
          view.Collection = Collections.News;
          return view;
      }
}

View-ul N2N (mulți-la-mulți)

Exemplu:

O știre/categorie de știri are mai multe tag-uri, iar un tag este conținut în mai multe știri

public Foundation.MVC.Views.N2NView TagsView
{
    get
    {
        Foundation.MVC.Views.N2NView view = new Foundation.MVC.Views.N2NView();
        view.Identification = "CategoryTags";
        view.Caption = "Category Tags";
        view.CollectionN2N = true;
        view.LReference = "NewsCategory";
        view.RReference = "Tag";
        view.ObjectN2N = typeof(NewsCategoryXTag).FullName;
        return view;
    }
}

Proprietăți care au fost setate pentru configurarea view-ului:

CollectionN2N - tipul de colectie (n - n);
LReference - referinta către obiectul din stanga (Left);
RReference - referinta către obiectul din dreapta (Right);
ObjectN2N = typeof(NewsCategoryXTag).FullName -

View-ul Usercontrol (Obsolete)

Din categoria view-urile avem si Usercontrol:

Exemplu:

protected Foundation.MVC.Views.View Designer
{
    get
    {
        Foundation.MVC.Views.UserControlView view = new Foundation.MVC.Views.UserControlView();
        view.Path = "/u/ModelDesigner.ascx";
        view.Category = "Designer";
        view.Caption = "<i class='icon-magic-wand'></i> Designer";
        view.Identification = "Designer";
        view.Initialized += new EventHandler <Foundation.MVC.Views.UserControlViewInitializeArgs>(OnDesignModelViewQuestionnaireInitialized);
        view.VisibleOnNew = false;
        return view;
    }
}

Exemplu:

void OnDesignModelViewQuestionnaireInitialized(object sender, Foundation.MVC.Views.UserControlViewInitializeArgs e)

     CatalogEntry.CreateModelTemplate();

}

Model View

Este folosit pentru a încărca un model într-un alt model. Spre exemplu, în News va fi încărcat Model Contact. Între News și Contact nu este nici-o legătură.
Pentru a invalida instanța model: Model.ClearCurrentEditingModel().

Exemplu:

public Foundation.MVC.Views.ModelView ModelView
{
    get
    {
    Contact contact = Q.Object.Lookup.Lookup.Get(Contact.Type).FirstOrDefault<Contact>();
    Foundation.MVC.Views.ModelView modelView = new Foundation.MVC.Views.ModelView();
    modelView.Model = Foundation.MVC.Model.GetInstanceModel(contact);
    modelView.Caption = "Model contact";
    modelView.Identification = "modelcontact";
    modelView.Instance = contact;

    return modelView;
    }
}

ViewLookup

Este folosit asemeni unui results pe baza unui lookup. De exemplu, adăugarea în News a lookup-ului pe Contact.

Exemplu:

public Foundation.MVC.Views.ViewLookup Contacts
{
    get
    {
    Foundation.MVC.Views.ViewLookup view = new Foundation.MVC.Views.ViewLookup();
    view.Lookup = Q.Object.Lookup.Lookup.Get(Contact.Type);
    view.Identification = "Contacts";
    view.Caption = "<i class='icon-users'></i> Contacts";
                    
    return view;
    }
}
