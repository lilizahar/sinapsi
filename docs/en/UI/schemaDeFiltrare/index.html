Schema de filtrare

Sumar

Schemele de filtrare sunt declarate în partea UI a unei clase. O schema de filtrare este metodata statica care este decorata cu atributul Q.Object.Attributes.FilterSchema și returnează un obiect de tipul Q.Object.UI.Controls.Filter.Schema. Dacă atributul FilterSchema nu are setat proprietatea Name atunci acea schema de filtrare va fi schema predefinita.

Exemplu:

[Q.Object.Attributes.FilterSchema()]
public static Q.Object.UI.Controls.Filter.Schema SchemaFilter()
{
    Q.Object.UI.Controls.Filter.SchemaBuilder builder = new Q.Object.UI.Controls.Filter.SchemaBuilder(Type);

    builder.Add(Elements.Login, Q.Resources.Strings.Login);
    builder.Add(Elements.Name, Q.Resources.Strings.Name);
    builder.AddDropDownList(Elements.Role, Q.Resources.Common.UserRole);
    builder.Add(Elements.LastLogin, Q.Resources.Strings.LastAccess);
    builder.Add(Elements.Notes, Q.Resources.Strings.Notes);

    return builder.Schema;
}

Builder

Construcția unei scheme de filtrare se va face prin intermediul clasei de suport Q.Object.UI.Controls.Filter.SchemaBuilder. SchemaBuilder are diferite metode de a adăuga elemente în schema de editare. Metoda Add va adăuga un control apropiat tipului proprietății.

Tipuri de elemente de filtrare

Metoda Add va adăuga un control apropiat tipului proprietății, dar se poate preciza și explicit ce element va fi folosit pentru o proprietate. Declararea explicită tipului de element poate fi făcută prin chemarea metodei apropiate controlului (cum ar fi pentru un dropdownlist, AddDropDownList) sau prin adaugarea directa a controlului specific.

Exemplu:

În următorul exemplu se va declara explicit pentru proprietatea RefCasualeContabile folosirea filtrului de tip ItemReferenceById.

Q.Object.UI.Controls.Filter.T0x02.ItemReferenceById causaleContabile = new Q.Object.UI.Controls.Filter.T0x02.ItemReferenceById(Elements.RefCausaleContabile, CausaleContabile.Elements.Description);
causaleContabile.DisplayName = Q.Resources.Resources.GetString("$CausaleContabile");
causaleContabile.OverlayEdit.DataTextField = String.Empty;
builder.Add(causaleContabile);

Corespondenta elemente de filtrare si lookup

La filtrarea unui results în funcție de selecția elementelor din schema de filtrare, se va pasa lookup-ul specific gridului cu filtrele setate din schema de filtrare.

Initial lookup

Unei scheme de filtrare i se poate asocia un inițial lookup, acest lookup va prefiltra grid-ul la incarcare.

Q.Object.UI.Controls.Filter.Schema schema = builder.Schema;
schema.InitialLookup = Q.Object.Lookup.Lookup.Get(User.Type).AddFilter(true, Elements.Enabled);

Evenimentul de inițializare

Evenimentul de inițializare este folosit cel mai des în a schimba comportamentul elementelor. Comportamentele care se schimbă des sunt vizibilitatea, readonly, additionallookup pentru elementele care suporta additional lookup.

Sintaxa declarativă a evenimentului de inițializare

Q.Object.UI.Controls.Filter.Schema schema = builder.Schema;           
schema.Initialize += OnSchemaFilterInitialize;

Exemplu 1:

În următorul exemplu proprietatea Name va fi Readonly iar proprietatea Code va fi ascunsă.

private static void OnSchemaFilterInitialize(object sender, Q.Object.UI.Controls.Filter.InitializeArgs e)
{
    e.Schema.GetItemByProperty(new Q.Object.Lookup.Target(Elements.Code)).Visible = false;
    e.Schema.GetItemByProperty(new Q.Object.Lookup.Target(Elements.Name)).ReadOnly = false;
}

Exemplu 2:

În următorul exemplu pentru proprietatea role se seteaza un additional lookup.

private static void OnSchemaFilterInitialize(object sender, Q.Object.UI.Controls.Filter.InitializeArgs e)
{
    ((Q.Object.UI.Controls.Base.IAdditionalLookup)e.Schema.GetItemByProperty
        (new Q.Object.Lookup.Target(Elements.Role))).AdditionalLookup
        = Q.Object.Lookup.Lookup.Get(Role.Type).AddFilter(false,Role.Elements.Administrator);
}

Evenimentul de schimbare a unei proprietăți

De fiecare data când evenimentul de schimbare a unei proprietăți se declanșează , se va declanșa și evenimentul de inițializare schemei de editare.
Evenimentul de schimbare a unei proprietăți este folosit atunci cand pentru o anumită proprietate la schimbarea valorii vrem sa modificăm comportamente pentru alte proprietăți din schema de filtrare care depind de selecție.

Sintaxa declarativă a evenimentului de schimbare a unei proprietăți

Q.Object.UI.Controls.Filter.Schema schema = builder.Schema;
schema.Rules[new Q.Object.Lookup.Target(Elements.Assestment)].Changed += OnAssestmentChanged;
private static void OnAssestmentChanged(object sender, Q.Object.UI.Controls.Filter.RuleArgs e)
{
}

Schema brief

Schema brief este o schema de filtrare care, în atributul decorativ, va avea setat flag-ul Brief cu valoarea true. Schema brief este o schema de filtrare simpla care va afișată tot timpul în pagina, in schimb schema de filtrare normala trebuie apăsat butonul de aditional filter pentru a afișa schema de filtrare în pagina.

În urmatoarea imagine avem schema brief folosită pentru User, iar în dreapta se poate observa butonul pentru additional filter.
