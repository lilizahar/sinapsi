Schema de results

Sumar

Schemele de rezultate sunt declarate în partea UI a unei clase. O schema de results este metodata statica care este decorata cu atributul Q.Object.Attributes.ResultSchema și returnează o un obiect de tipul Q.Object.UI.Controls.Grid.Schema. Dacă atributul ResultSchema nu are setat proprietatea Name atunci acea schema de results va fi cea predefinita.

Exemplu:

[Q.Object.Attributes.ResultSchema()]
public static Q.Object.UI.Controls.Grid.Schema SchemaResults()
{
    Q.Object.UI.Controls.Grid.SchemaBuilder builder = new Q.Object.UI.Controls.Grid.SchemaBuilder(Type);

    builder.Add(Q.Resources.Strings.Code, System.Web.UI.WebControls.Unit.Percentage(10), Elements.Code);
    builder.Add(Q.Resources.Strings.Login, System.Web.UI.WebControls.Unit.Percentage(20), Elements.Login);
    builder.Add(Q.Resources.Strings.Name, System.Web.UI.WebControls.Unit.Percentage(25), Elements.Name);
    builder.Add(Q.Resources.Strings.Role, System.Web.UI.WebControls.Unit.Percentage(15), Elements.Role, Role.Elements.Name);
    builder.Add(Q.Resources.Strings.Activ, System.Web.UI.WebControls.Unit.Percentage(10), Elements.Enabled);
    builder.Add(Q.Resources.Strings.LastAccess, System.Web.UI.WebControls.Unit.Percentage(5), Elements.LastLogin);
    builder.Add(Q.Resources.Strings.Notes, System.Web.UI.WebControls.Unit.Percentage(15), Elements.Notes);

    builder.SetNavigateUrl(String.Format(@"{0}&Id={{ID}}", Foundation.Objects.Object.EditPage(Type)));
    builder.MultipleSelection = true;
    builder.Selectable = true;

    return builder.Schema;
}

Results vs Reference

Schemele de results sunt afișate în listările directe ale obiectelor, dar atunci cand va fi accesat un results al unei clase, spre exemplu din contextul unui item de timp reference, se va încărca schema reference. Schema reference este o schema de results, dar în plus acea schema de results atributul decorativ va avea proprietatea ReferenceSchema setat pe true.

Exemplu:

[Q.Object.Attributes.ResultSchema(ReferenceSchema = true)]
static public Q.Object.UI.Controls.Grid.Schema SchemaReference()
{
     Q.Object.UI.Controls.Grid.SchemaBuilder builder = new Q.Object.UI.Controls.Grid.SchemaBuilder(Type);

     builder.Add(Q.Resources.Strings.Name, System.Web.UI.WebControls.Unit.Percentage(70), Elements.Name);
    builder.Add(Q.Resources.Strings.Role, System.Web.UI.WebControls.Unit.Percentage(30), Elements.Role, Role.Elements.Name);

    builder.Selectable = true;
    builder.MultipleSelection = true;

    return builder.Schema;
}

Builder

Construcția unei scheme de results se va face prin intermediul clasei de suport Q.Object.UI.Controls.Grid.SchemaBuilder. SchemaBuilder are diferite metoda de a adăuga elemente în schema de editare. Metoda Add va adăuga un control apropiat tipului proprietății.

Evenimentul de inițializare

Evenimentul de inițializare este folosit pentru a customiza gridul afișat.

Exemplu:

În următorul exemplu, pentru liniile în care instanța are flagul Cancelable pe false, background-color va fi setat cu #f1f1f1.

Q.Object.UI.Controls.Grid.Schema schema = builder.Schema;
schema.Initialize += new EventHandler<EventArgs>(OnGridInitialize);
return schema;

static void OnGridInitialize(object sender, EventArgs e)
{
    ((DataGrid)sender).ItemDataBound += new DataGridItemEventHandler(OnDocumentItemDataBound);
}

static void OnDocumentItemDataBound(object sender, DataGridItemEventArgs e)
{
    if (e.Item.ItemType == ListItemType.Item ||
        e.Item.ItemType == ListItemType.AlternatingItem)
    {
        if (!((DocumentItem)e.Item.DataItem).Cancelable)
            e.Item.Style.Add("background-color", "#f1f1f1");
    }
}

Tipuri de coloane

După cum știm anterior, metoda Add va adăuga un control apropiat tipului proprietății, dar se poate preciza și explicit ce element va fi folosit pentru o proprietate. Declararea explicită tipului de element poate fi făcută prin chemarea metodei apropiate controlului (cum ar fi pentru un tip de date numeric, Add Numeric) sau prin adaugarea directa a controlului specific. Coloanele în schemele de results pot fi editabile.

Exemplu 1:

În următorul exemplu se va adăuga o coloana de tip checkbox pentru proprietatea UIEnabled.

builder.AddEditableCheckBox("UIEnabled", System.Web.UI.WebControls.Unit.Percentage(10), Elements.UIEnabled);

Exemplu 2:

În următorul exemplu se va adăuga o coloana de tip hyperlink pentru proprietatea Read.

Q.Object.UI.Controls.Grid.Item.Hyperlink hyperlink = new Q.Object.UI.Controls.Grid.Item.Hyperlink(new Q.Object.UI.Controls.Grid.Item.Hyperlink.CallbackHRef(OnHyperlinkCallbackNewsletter));
Q.Object.UI.Controls.Grid.TemplateColumn hyperlinkColumn = new Q.Object.UI.Controls.Grid.TemplateColumn
(
     new Q.Object.UI.Controls.Grid.Header.Label(String.Empty), hyperlink, Unit.Pixel(24), Elements.Read
);
builder.Add(hyperlinkColumn);

Exemplu 3:

public static string OnHyperlinkCallbackNewsletter(object sender, Q.Object.Object instance)
{
    ((HyperLink)sender).CssClass = "external1";
    ((HyperLink)sender).Text = "&nbsp;";
    ((HyperLink)sender).Target = "_blank";

    return                 
      String.Format("/Admin/Edit.aspx?T=Mailing.Objects.Newsletter&M=Newsletter&Id={0}",           
                     ((NewsletterRecipient)instance).Newsletter.ID);
}

Selectable

După cum am observat în exemplele anterioare pentru schemele de results în builder putem seta proprietatea Selectable. Setarea lui Selectable pe false va dezactiva posibilitatea de a intra în schema de editare a liniei pe care se va face click.

MultipleSelection

Pentru schemele de results în builder putem seta proprietatea MultipleSelection. Setarea lui MultipleSelection pe false va dezactiva posibilitatea de a selecta linii pentru a se face acțiuni multiple, cum ar fi cea de a șterge liniile selectate din grid.

Exemplu:

În următorul exemplu avem o schema de results cu MultipleSelection pe true; Dacă se va seta MultipleSelection pe false, checkbox-ul de la începutul liniei va dispărea.

Navigation

Pentru schemele de results în builder putem seta unde va naviga la click-ul unei linii prin metoda SetNavigateUrl.

Scheme de results multiple

Se pot defini mai multe scheme de results în contextul aceleași clase. Schemele trebuie decorate cu atributul Q.Object.Attributes.ResultSchema, iar pentru celelalte scheme care nu sunt default trebuie setat atributul Name.

Exemplu:

[Q.Object.Attributes.ResultSchema(Name = "Details")]
public static Q.Object.UI.Controls.Grid.Schema SchemaReferenceDetails()
