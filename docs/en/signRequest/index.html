SignRequest

  • accepts only one document to sign (can be implemented a document chain to be sent with previous_document, starting with the last one and going back to the first, signing each document)
  • multiple signers (even with the same details)
  • the signing log is sent as a separate document, containing all of the signers details
  • the from email and the document link are set in CreateSessionCore() with the keys from SignFromEmail and the SignRequestDocumentLink(created with the account creation) are set in web.config
  • the authority is configured in SignRequestAuthority() with the token and the key prefix created with the creation of the account (those are set in web.config)
  • a canceled request is still available for download and check

See more: https://signrequest.com/en
