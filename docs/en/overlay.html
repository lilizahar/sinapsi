Overlay

Sumar

La momentul selecției checkbox-ului Edit in overlay din setările Sinapsi, pagina de editare se deschide într-o fereastră nouă de tip overlay (suprapunere). Dacă se dorește personalizarea acestei pagini se vor aborda pași similari cu cei descriși în User Control, unde se găsesc și controalele specifice editării în overlay. Overlay-urile predefinite sunt Results (Foundation.U.Results) și Edit (Foundation.U.MCV.Edit).

OverlayBase (din Foundation)

Exemplu:

Un exemplu relevant pentru editarea overlay este duplicarea. În momentul în care este apăsat butonul Duplicate (adăugat în secțiunea EditInfo) se creează o copie a elementului (copie deschisă în mod overlay), care însă exclude o categorie (tag-urile). În figura de mai jos este deschisă în editare o știre.

După apăsarea butonului Duplicate se deschide în overlay o copie a știrii anterioare, copie care nu reține tag-urile. Numele noii știri este o copie a anteriorului, la care se adaugă structura “(copy)”.

Pentru a se realiza acest lucru s-a procedat în felul următor:

  • în EditInfo s-a adăugat butonul Duplicate

  • butonul Duplicate are atașată în fișierul .cs o funcție OnDuplicate care:

    • creează o instanță a clasei ce urmează a fi duplicată
    • apelează metoda Duplicate din clasă
    • construiește argumentele pentru overlay:
      • argument1 = type
      • argument2 = identification
      • argumentele pentru overlay sunt organizate sub formă de dicționare <string, object>
      • aceste argumente sunt necesare pentru a putea construi instanța Overlay prin (Q.BaseClasses.Page)Page).Overlay("~/Admin/U/Prototypes/EditOverlay/Overlay.ascx", arguments, UniqueID)

  • metoda Duplicate din clasa de bază a cărei instanțe este copiată se descrie astfel

    • clasa Cloner instanțiată prin intermediul cloner
    • la clonare este important de reținut faptul că se copiază și colecțiile asociate sub forma unei structuri arborescente (se clonează întâi nodul părinte, se caută apoi copii <vezi referință ClonedQueue>)
    • cea mai sugestivă funcție din clasa Clone este CloneCore care:
      • identifică elementul ce urmează a fi clonat
      • îl adaugă în coadă
      • citește din Storage elementele ce se copiază prin DataGetImpl și le salvează prin DataSetImpl în storage-ul copiei
      • relații one2one

O informație importantă de reținut este următoarea: orice obiect Q.Object.Object este format din Schema de stocare și Storage (sub formă de hashtable). Toate operațiile Load și Save din/în baza de date sunt realizate cu ajutorul Storage. Secvența Load este Select -> Load -> Populate Storage, iar secvența Save Storage -> Update/Insert -> Sql -> Execute. De asemenea, când se face verificarea Dirty se folosește tot Storage.

EditOverlay.Helper

Exemplu:

Foundation.U.Prototype.EditOverlay.Helper.Add(Local.Objects.Contact.Type)
                    .AddArgument("CloseOverlayAfterSave", true)
                    .AddInvariant("Name", UniqueID)
                    .AddInvariant("Team", team.ID)
                    .AddInvariant("Primary", "False")
                    .AddInvariant("Surname", "valoare predefinita")
                    .AddInvariant("Email", UniqueID)
                    .Open(UniqueID);

În momentul deschiderii în overlay a paginii de creare Contact, prin adăugarea acestor proprietăți Invariant, câteva (Name, Team, Primary, Surname, Email) câmpuri vor avea valori predefinite.

Metoda AddArgument adaugă un argument specific, în cazul de față, la salvarea unei instanțe noi în baza de date, pagina deschisă în overlay se va închide.

Metoda Open(UniqueID) se folosește deoarece fiecare entitate deschisă are un identificator cu care se diferențiază de celelalte (+în momentul deschiderii editării în Overlay trebuie reținut ID-ul instanței). Spre exemplu, într-un overlay se deschide un alt overlay. În momentul închiderii overlay-ului este necesar a ști unde revenim sau unde ar trebui să se revină.

Evenimente:

void OnOverlayClosed(object sender, EventArgs args)
{
    Foundation.U.Prototype.EditOverlay.EditOverlayClosedArgs editingArgs = args as Foundation.U.Prototype.EditOverlay.EditOverlayClosedArgs;
    if (editingArgs != null && editingArgs.StartArguments.ContainsKey("MyArgument"))
    {     
        HostPage.PostStartUpMessage(editingArgs.StartArguments["MyArgument"].ToString());
    }
}
