Menținerea coerenței bazei de date

Sumar

O bază de date consistentă este o BD care, la momentul unei tranzacții, transformă datele într-o manieră stabilită.

Exemplu: News, NewsCategory, Tag

O știre are o singură categorie, dar mai multe taguri.

În momentul în care se dorește ștergerea unei știri, apare o problemă: instanța este referită în alte locuri, deci nu poate fi ștearsă.

Pentru a soluționa această problemă trebuie menționat faptul că în momentul unei ștergeri, se șterg și dependințele, adică toate liniile unde știrea este referită.

Acest lucru se poate realiza în două moduri: * din cod prin suprascrierea metodei Delete(), specificând și ștergerea asocierilor * din baza de date (vezi următoarea imagine)

Cascade -> Toate elementele referite vor fi șterse

Set Null -> Toate elementele referite vor fi trecute pe Null (Importante de verificat sunt constrângerile de pe coloana referită, este necesar să accepte valori nule)
